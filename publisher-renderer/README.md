# coruro-publisher-renderer

This is Coruro's **4th process surface** — a Chromium-bearing, local-only Node + Playwright renderer that turns the app's Neo-Brutalist cards into PNG carousel pages for the Publisher. It is spawned over stdio by `src-tauri/src/publisher.rs` (reads one JSON line `{repo,target,data,outDir}` on stdin, prints `{assets:[...absolute paths]}` on stdout), loads the offscreen React build at `../dist/offscreen.html` over `file://`, and screenshots each `.coruro-card-page`. It is explicitly **NOT** the zero-network FoundationModels AI sidecar (`ai-sidecar/`) — it does no AI work and loads only `file://`/`127.0.0.1` origins. It is kept as a separate npm package so its Playwright/Chromium dependency never pollutes the root gate or CI (excluded like the Swift sidecar). Activate it with: `cd publisher-renderer && npm install && npx playwright install chromium`.

> **Known limitation:** rendering is opt-in and spawns `node` via the inherited `PATH`. A Finder-launched release bundle may not inherit a shell `PATH`, so image export currently requires launching Coruro from a shell-inherited environment (e.g. `open` from a terminal, or `npm run tauri dev`); a renderer-absent run degrades gracefully to text-only drafts. We do not bundle `node`.
